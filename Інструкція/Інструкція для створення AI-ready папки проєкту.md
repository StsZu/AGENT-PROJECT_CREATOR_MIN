**Інструкція для створення AI-ready папки проєкту**

Мета: створити папку проєкту так, щоб її могли швидко читати й оновлювати AI-агенти, CLI-агенти та чати ШІ без зайвого перетворення документів.

1. Створи окрему папку проєкту

Назва папки має бути короткою, зрозумілою і без зайвих символів.

Приклади:

```text
client-crm-audit
website-redesign-2026
sales-process-docs
warehouse-automation
```

Краще використовувати:

```text
lowercase-words-with-dashes
```

Не бажано:

```text
Новий проєкт фінальний 2
Project!!! copy
Документи різне
```

2. Використовуй текстові формати файлів

Основні рекомендовані формати:

```text
.md
.txt
.csv
.json
.yaml
.yml
.toml
```

Головний формат для документації — `.md`.

Використовуй `.md` для:

```text
опису проєкту
вимог
інструкцій
задач
рішень
ризиків
нотаток
зустрічей
опису процесів
```

Використовуй `.csv` для таблиць:

```text
списки клієнтів
реєстри
таблиці статусів
каталоги
експорти з CRM
```

Використовуй `.json`, `.yaml`, `.yml` або `.toml` для структурованих даних і конфігів.

3. Не використовуй важкі формати як основне джерело правди

Не роби основними документами:

```text
.docx
.pdf
.pages
.xlsx
скріншоти з текстом
фото документів
```

Їх можна зберігати як додаткові матеріали, але основний зміст треба перенести в `.md`, `.txt` або `.csv`.

4. Створи базову структуру папки

Рекомендована мінімальна структура:

```text
project-name/
  README.md
  CONTEXT.md
  REQUIREMENTS.md
  TASKS.md
  DECISIONS.md
  RISKS.md
  NOTES.md
  ACCESS.example.md
  data/
    README.md
  docs/
    00-README.md
    01-domain.md
```

5. Заповни `README.md`

Це головний файл проєкту. У ньому має бути коротка відповідь на питання: що це за проєкт і навіщо він існує.

Шаблон:

```md
# Назва проєкту

## Мета

Коротко описати, для чого існує проєкт.

## Поточний стан

Що вже є, що ще не зроблено.

## Важливі файли

- `CONTEXT.md` — загальний контекст
- `REQUIREMENTS.md` — вимоги
- `TASKS.md` — задачі
- `DECISIONS.md` — прийняті рішення
- `RISKS.md` — ризики
```

6. Заповни `CONTEXT.md`

Тут треба описати фон проєкту.

```md
# Context

## Що відбувається

Опис ситуації.

## Для кого цей проєкт

Користувачі, клієнти або команда.

## Обмеження

Терміни, бюджет, технічні або організаційні обмеження.

## Що важливо знати AI-агенту

Короткі правила, факти, заборони або домовленості.
```

7. Заповни `REQUIREMENTS.md`

Тут зберігай вимоги.

```md
# Requirements

## Обов’язково

- ...

## Бажано

- ...

## Не робити

- ...
```

8. Веди `TASKS.md`

Файл для задач.

```md
# Tasks

## Active

- [ ] P0-001 Опис задачі
- [ ] P1-001 Опис задачі

## Done

- [x] P0-000 Початкове створення структури
```

Рекомендовані пріоритети:

```text
P0 — критично
P1 — важливо
P2 — бажано
P3 — колись
```

9. Веди `DECISIONS.md`

Сюди записуй важливі рішення, щоб агент не питав одне й те саме повторно.

```md
# Decisions

## YYYY-MM-DD — Назва рішення

Рішення:

Причина:

Наслідки:
```

10. Веди `RISKS.md`

```md
# Risks

| ID | Ризик | Ймовірність | Вплив | Що робимо |
|---|---|---:|---:|---|
| R-001 | Опис ризику | Medium | High | План дій |
```

11. Використовуй `ACCESS.example.md`, але не зберігай секрети

У цьому файлі можна описати, які доступи потрібні, але без паролів і токенів.

```md
# Access Example

## Потрібні доступи

- CRM
- Hosting
- Analytics
- Git repository

## Де отримати

Описати відповідального або процес.
```

Не записуй у папку:

```text
паролі
токени
приватні ключі
.env з реальними секретами
```

12. Додай `data/` для таблиць і структурованих даних

```text
data/
  README.md
  clients.csv
  products.csv
  settings.example.yaml
```

У `data/README.md` коротко поясни, що лежить у папці.

13. Додай `docs/` для розширеної документації

```text
docs/
  00-README.md
  01-business-process.md
  02-technical-notes.md
  03-meetings.md
```

Нумерація допомагає агентам і людям читати документи в правильному порядку.

14. Правила для назв файлів

Добре:

```text
README.md
TASKS.md
DECISIONS.md
01-business-process.md
clients.csv
settings.example.yaml
```

Погано:

```text
Фінал новий останній.docx
таблиця копія 3.xlsx
скан документа.jpg
Без названия.txt
```

15. Перед передачею папки AI-агенту перевір

У папці має бути:

```text
README.md
CONTEXT.md
REQUIREMENTS.md
TASKS.md
DECISIONS.md
RISKS.md
```

Основні файли мають бути в `.md`, `.txt`, `.csv`, `.json`, `.yaml`, `.yml` або `.toml`.

Не повинно бути секретів, паролів, токенів або приватних ключів.

Коротке правило для співробітника:

**Якщо документ має читати AI-агент, створюй його як `.md`. Якщо це таблиця — `.csv`. Якщо це конфіг або структуровані дані — `.yaml` або `.json`. PDF, DOCX і XLSX використовуй тільки як додаткові вкладення, не як основну документацію.**
