# TASKS

Операційна дошка задач. Не вільний список ідей — структурований протокол виконання.

---

## Rules

- Виконуються тільки задачі зі `Status: READY` **і** `Safety: READ-ONLY`.
- Все інше (`WRITE`, `RISKY`, `BLOCKED`, `APPROVAL`) — потребує явного підтвердження людини, не "на сьогодні", а на кожну окрему задачу.
- Задача без поля `Done when:` — НЕ задача. Спершу заповнити критерій, потім починати.
- Пріоритет жорстко фіксований: P0 виконується першим, P1 — тільки коли всі P0 у статусі `DONE` або `BLOCKED`.

## Legend

**Status:**
- `[ ]` READY — готово до виконання
- `[~]` IN PROGRESS — хтось працює, не чіпати
- `[x]` DONE — виконано, є артефакт
- `[!]` BLOCKED — чекає на залежність / approval
- `[A]` APPROVAL — потребує одноразового дозволу людини

**Safety:**
- `READ-ONLY` — не змінює стан системи. Безпечно.
- `WRITE` — змінює локальні файли проєкту. Помірний ризик.
- `RISKY` — важко відкотити (deploy, delete, drop). Тільки під approval.

## Current Execution Mode

Дивись [SESSION.md](SESSION.md) → `Current Execution Mode`. Це єдине джерело правди про поточний режим — не дублювати тут.

## Global Prohibitions

- Не запускати команди з `--force`, `-rf`, `--no-verify` без явного approval.
- Не комітити `ACCESS.md`, `.env`, `*.key`.
- Не змінювати конфіги production без `WRITE` режиму + `Done when` + rollback plan.

## Done (append-only ledger)

Кожен виконаний крок отримує ID `D-NNN` і вказує на артефакт. Не редагувати, тільки додавати.

| ID | Дата | Task | Артефакт |
|---|---|---|---|
| D-001 | YYYY-MM-DD | P0-001 | `<шлях до файлу або вивід команди>` |

## Errata

| Дата | Помилка | Корекція |
|---|---|---|
| YYYY-MM-DD | <що було неправильно> | <що зроблено> |
