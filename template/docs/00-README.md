# docs/

Доменні референси проєкту. Один файл = один великий об'єкт / підсистема.

## Конвенції іменування

- `01-<домен>.md`, `02-<домен>.md`, … — порядкові номери, щоб у `ls` був стабільний порядок читання.
- Назва домену — англійською, lowercase, з тире (`01-infrastructure-map.md`, не `01-Карта.md`).
- Один файл — один об'єкт. Не змішувати "UPS + бекапи + мережа" в один документ.

## Що йде сюди

- Інвентаризація обладнання / сервісів.
- Топологічні мапи (мережа, потоки даних, залежності).
- Снапшоти стану на конкретну дату.
- Per-вузол / per-сервіс звіти.

## Що НЕ йде сюди

- Задачі → [TASKS.md](../TASKS.md)
- Рішення → [DECISIONS.md](../DECISIONS.md)
- Ризики → [RISKS.md](../RISKS.md)
- Поточна сесія → [SESSION.md](../SESSION.md)
- Пояснення для замовника → [simple_words.md](../simple_words.md)

## Правила оновлення

- На початку файлу: `**Last updated:** YYYY-MM-DD` + джерело даних (команда / артефакт).
- Кожен факт — з посиланням на артефакт (`/tmp/discovery/foo.out`, `git log <hash>`, скріншот).
- Поля `UNKNOWN` / `NOT VERIFIED` — обов'язкові, коли інформації немає. Не вигадуємо.
