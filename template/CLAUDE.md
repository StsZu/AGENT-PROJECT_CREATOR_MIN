## Default mode

`<READ-ONLY | WRITE | RISKY>` — поки явно не змінено в [SESSION.md](SESSION.md) → `Current Execution Mode`.

- **READ-ONLY** — дозволено лише читати файли, запускати безпечні команди (без `--write`, без `rm`).
- **WRITE** — дозволено редагувати локальні файли проєкту.
- **RISKY** — дозволено зміни, які важко відкотити (deploy, видалення, зовнішні побічні ефекти). Тільки під явний одноразовий approval, не "на весь день".

## Що дозволено без додаткового дозволу

- Читати будь-який файл у папці проєкту.
- Запускати read-only команди (`ls`, `grep`, `cat`, `find`, `wc`).
- Створювати нові файли у власних робочих директоріях (`docs/`, `logs/`, `scratch/`).
- Оновлювати [SESSION.md](SESSION.md), [TASKS.md](TASKS.md) (статуси), [DECISIONS.md](DECISIONS.md) (нові записи).

## Що заборонено без явного одноразового approval

- `rm -rf`, масове видалення або перезапис файлів.
- Зміна паролів, токенів, секретів.
- Будь-яка дія з `Safety: RISKY` у [TASKS.md](TASKS.md).
- Виконання задач зі `Status: BLOCKED` або `Status: APPROVAL`.

## Що можна чіпати, що ні (поточний стан)

**Можна редагувати без approval:**
- [SESSION.md](SESSION.md), [TASKS.md](TASKS.md) (статуси), [DECISIONS.md](DECISIONS.md), [RISKS.md](RISKS.md) — append-only або статус-апдейти.
- Нові файли у `docs/`, `logs/`, `scratch/`, які створює ця ж сесія.
- Власноруч створені артефакти (вивід тестів, скріншоти, JSON-дампи).

**Не редагувати без явного запиту користувача:**
- [README.md](README.md), [CLAUDE.md](CLAUDE.md), [simple_words.md](simple_words.md), [DEMO.md](DEMO.md) — встановлюються на bootstrap, переписуються тільки коли проєкт змінює суть.
- Numbered domain files (`docs/01-*.md`, `docs/02-*.md`) — авторський контент.
- Файли, які прийшли з legacy-проєкту під час Mode B міграції.

**Ніколи не видаляти напряму:**
- Не використовувати `rm` або `rm -rf` без approval.
- Сміття, дублі, забуті чернетки — **переміщувати в `_attic/YYYY-MM-DD/`** через `mv` після approval і запису в `DECISIONS.md`. Видалення `_attic/` — окремий крок з окремим approval після того, як проєкт пройшов `DEMO.md`.
- OS-сміття (`.DS_Store`, `Thumbs.db`, `*~`, `*.bak`) — не видаляти руками без approval.

**Карантин (`_attic/`) — правила:**
- Структура: `_attic/YYYY-MM-DD/<original-path>` — зберігаємо підшлях, щоб потім легко повернути.
- Не лишати `_attic/` назавжди — або cleanup після sign-off з approval, або залишити як архівний шар з поясненням у `START_HERE.md`.

## Формат фактів

У всіх документах використовувати три рівні впевненості:

- **факт** — підтверджено власноруч, є артефакт (вивід команди, скріншот, файл).
- **`NOT VERIFIED`** — інформація з документації / усного джерела, але не перевірено емпірично.
- **`UNKNOWN`** — поле порожнє, інформації немає, припущень не робимо.

**Заборонено вигадувати значення.** Якщо невідомо — пишемо `UNKNOWN`, а не "ймовірно", "схоже на", "має бути".

## Пріоритет джерел знань

Локальні файли цього проєкту мають **пріоритет над auto-memory** Claude Code чи будь-якою глобальною пам'яттю агента. Якщо memory суперечить файлу — оновлювати файл (це source of truth), а не покладатись на memory.

## Перед виконанням будь-якої задачі

1. Прочитати поточний `Current Execution Mode` у [SESSION.md](SESSION.md).
2. Прочитати поле `Safety:` задачі. Якщо `RISKY` — зупинитись і дочекатись підтвердження.
3. Прочитати поле `Done when:` задачі. Якщо порожнє — НЕ починати, спершу заповнити критерій.
4. Прочитати `Depends:` — переконатись, що залежні задачі мають статус DONE.

## Після виконання задачі

1. Записати результат у [SESSION.md](SESSION.md) — що зроблено, які артефакти, де лежать.
2. Оновити статус задачі в [TASKS.md](TASKS.md): `[ ]` → `[x]`, додати запис у `Done` ledger (`D-NNN`).
3. Якщо знайдено нову проблему — створити нову задачу в `TASKS.md` зі статусом `READY` або `BLOCKED`.
