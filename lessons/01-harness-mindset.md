# Урок 1. Harness mindset

**Мета курсу:** навчити створювати універсальний AI-ready проєкт для будь-якої задачі — зі структурою папок і operational harness.

## Що таке harness

Harness — операційна рамка навколо задачі:

- структура папки (що де лежить);
- ритуали сесії (як починати і закривати роботу);
- backlog (`TASKS.md`);
- режими безпеки (READ-ONLY / WRITE / RISKY);
- sign-off (`DEMO.md` + Done when).

Це не «ще один README». Це система, яка тримає проєкт керованим разом з AI-агентом.

## MIN у двох реченнях

**MIN** = мінімальний набір документів + ритуал, щоб запустити проєкт за ≤10 хв і не накопичувати розсинхрон.

Універсально для: API, сайту, інфра, курсу, hardware, внутрішнього tooling.

## Ключові принципи

1. Один проєкт = одна папка.
2. Root = control room (не сміття, не raw exports).
3. Файли > пам'ять агента.
4. Default READ-ONLY.
5. Одна задача за сесію.
