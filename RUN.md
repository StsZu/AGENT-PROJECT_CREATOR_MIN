# Як використовувати

Два сценарії запуску. Обери відповідний, скопіюй одно-рядкову команду, встав у Claude CLI.

---

## Сценарій A — новий проєкт з нуля (Mode A)

У терміналі:

```bash
cd <PROJECTS_ROOT>     # або будь-яка папка-батько
claude
```

У чаті Claude:

```
Прочитай <KIT_PATH>/PROMPT.md і виконай Mode A. Цільова папка: <PROJECTS_ROOT>/<нова-назва>
```

---

## Сценарій B — приведення в порядок існуючого проєкту (Mode B)

У терміналі:

```bash
cd /path/to/legacy-project
claude
```

У чаті Claude:

```
Прочитай <KIT_PATH>/PROMPT.md і виконай Mode B на поточній папці.
```

---

## Що зробить агент

1. Прочитає 12 файлів зі стандарту ([STANDART.md](STANDART.md) → [METHODOLOGY.md](METHODOLOGY.md) → 10 файлів `template/`).
2. Поставить питання A1–A10 (Mode A) або B1–B3 (Mode B) — по одному, чекаючи відповідь.
3. Mode A — скопіює `template/.` у цільову папку і заповнить 5 файлів.
4. Mode B — виконає Прохід 1 (аудит, READ-ONLY) → STOP-gate → Прохід 2 (гепи) → STOP-gate → Прохід 3 (карантин у `_attic/`).
5. У кінці — summary "що зроблено / що далі".

---

## Якщо не Claude CLI (Gemini web, Cursor, ChatGPT, інше)

Скопіювати блок між `--- BEGIN ---` і `--- END ---` із [PROMPT.md](PROMPT.md), вставити у чат. Промпт самодостатній.

---

## Якщо щось пішло не так

- Зупинився посередині — безпечно почати спочатку. Без явного approval агент нічого не перезапише.
- Агент пропустив Крок 0 — нагадай: "перечитай Крок 0 у `PROMPT.md` перед першим питанням".
- Mode B застряг на Проході 1 — це by design. Жодних змін до явного "так, переходимо до Проходу 2".
- Папка з A2 уже існує — агент зупиниться і запитає. Не перезаписувати без розуміння, що там.
